# AGENTS.md

Этот репозиторий — портфолио-сайт на Vite + React + TypeScript.

## Старт работы

1. Убедись, что находишься в корне репозитория.
2. Прочитай `CURRENT.md` и проверь `git status -sb`.
3. Не перезаписывай пользовательские изменения и не расширяй scope без
   необходимости.
4. Работай над одной связной задачей за раз.

Полный baseline перед каждой правкой не обязателен. Сначала изучи только файлы,
которые относятся к запросу пользователя.

## Источники истины

- Публичные факты, тексты, ссылки и TODO находятся в `docs/content.md`. Читай
  этот файл перед изменением публичного текста и не придумывай отсутствующие
  факты, метрики, опыт, достижения или контакты.
- Структура, визуальный стиль, анимации и ассеты главной страницы описаны в
  `docs/site-brief.md`. Читай его перед соответствующими UI-изменениями.
- `CURRENT.md` — короткий snapshot незавершённой работы, а не журнал истории.
  Обновляй его только если работа остаётся незавершённой, заблокирована или
  следующий шаг нельзя восстановить из кода и Git.
- История завершённой работы хранится в Git.

Не используй логотип LEGO, официальные наборы, защищённых персонажей или чужие
брендированные ассеты. Сохраняй существующий русский текст, если пользователь
явно не просит его переписать. Не добавляй frontend-библиотеки без реальной
необходимости и понятного tradeoff.

## Карта проекта

- Точка входа: `src/main.tsx`.
- Маршруты `/`, `/employer` и `*`: `src/routes.tsx`.
- Страницы и общие стили: `src/pages/` и `src/pages/Page.module.css`.
- Глобальные стили: `src/styles/base.css` и `src/styles/tokens.css`.
- Контрактные тесты маршрутов: `src/routes.test.ts`.

## Проверка по риску

- Только Markdown: проверь diff, ссылки и устаревшие имена файлов.
- React/TypeScript: `npm run lint`, `npm run typecheck` и релевантные тесты.
- Маршруты или конфигурация: дополнительно `npm run build`.
- UI/CSS: дополнительно `npm run visual:smoke` и просмотри скриншоты.
- Значимые кодовые изменения или зависимости: `npm run check`.

Перед заявлением о готовности перечисли реально запущенные проверки. Для
незавершённой задачи оставь в `CURRENT.md` статус (`in_progress` или `blocked`),
scope, блокер и один следующий шаг. Для завершённой задачи достаточно статуса
`idle`; не добавляй накопительный отчёт.
