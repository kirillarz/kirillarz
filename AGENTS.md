# AGENTS.md

Этот репозиторий - портфолио-сайт на Vite + React + TypeScript. Codex должен
использовать этот файл как рабочий контракт при старте каждой новой сессии.

## Стартовый порядок работы (Startup Workflow)

Перед правками:

1. Убедись, что находишься в корне репозитория.
2. Прочитай `AGENTS.md`, `feature_list.json` и `progress.md`.
3. Проверь рабочее дерево через `git status -sb`; не перезаписывай изменения
   пользователя.
4. Прочитай `README.md` с командами проекта и `docs/content.md` перед
   изменением публичного текста портфолио.
   Перед изменением структуры, визуального стиля, анимаций или ассетов главной
   страницы прочитай `docs/site-brief.md`.
5. Запусти базовую проверку: `./init.sh` в Bash или `./init.ps1` в PowerShell.
   Если зависимостей нет, сначала выполни `npm ci`.
6. Выбери ровно одну незавершенную фичу из `feature_list.json`.

## Факты о проекте

- Точка входа приложения: `src/main.tsx`.
- Маршруты `/`, `/employer` и `*` описаны в `src/routes.tsx`.
- Страницы находятся в `src/pages/`.
- Общие стили страниц находятся в `src/pages/Page.module.css`.
- Глобальные CSS-файлы разделены на `src/styles/base.css` и
  `src/styles/tokens.css`.
- Контрактные тесты маршрутов находятся в `src/routes.test.ts`.
- Целевая Lego-style концепция и структура блоков описаны в
  `docs/site-brief.md`; hero-референсы лежат в `examples/`.

## Рабочие правила

- Правило One feature at a time: выбери ровно одну незавершенную фичу из
  `feature_list.json`; не начинай вторую, пока активная фича не получила статус
  `passing` или `blocked` с evidence.
- Держи изменения небольшими и связанными с одной активной фичей.
- Не придумывай факты портфолио, ссылки, метрики, опыт работы или достижения.
  Используй `docs/content.md`; неподтвержденные материалы помечай как `TODO`.
- Не придумывай визуальные требования сверх `docs/site-brief.md`; если данных
  не хватает, помечай `TODO` или фиксируй вопрос в `progress.md`.
- Не используй логотип LEGO, официальные наборы, защищенных персонажей или
  чужие брендированные ассеты; держи стиль как конструкторный/brick-toy
  inspired.
- Сохраняй существующий русский текст, если задача явно не просит его
  переписать.
- Обновляй тесты, если меняются маршруты или стабильное поведение.
- Не добавляй новые frontend-библиотеки без реальной необходимости и понятного
  tradeoff.
- Перед завершением сессии обнови `progress.md` и `feature_list.json`: статус,
  evidence, блокеры и следующий шаг.

## Граница scope (Scope Boundary)

- В scope: файлы и тесты, нужные для активной фичи, а также обновления состояния
  harness, которые сохраняют репозиторий restartable.
- Вне scope: несвязанные рефакторинги, новые зависимости, изменения деплоя и
  изменения контента портфолио, не подтвержденные `docs/content.md`.
- Если scope неясен, остановись и зафиксируй неоднозначность в `progress.md`
  перед расширением задачи.

## Проверка (Verification Commands)

Перед заявлением о готовности запусти полный набор проверок:

```bash
npm run lint
npm run typecheck
npm test
npm run build
```

`./init.sh` и `./init.ps1` запускают ту же последовательность.

Для визуальной smoke-проверки главной страницы используй `npm run visual:smoke`.
Команда запускает Playwright, поднимает или переиспользует Vite dev-server и
сохраняет скриншоты в `artifacts/home-desktop.png` и
`artifacts/home-mobile.png`; их можно открыть через `view_image`.

## Критерий готовности (Definition of Done)

Фича считается готовой только когда:

- Целевое поведение реализовано.
- Релевантные тесты или проверки добавлены либо обновлены.
- Полная проверка запущена, либо каждый пропущенный check объяснен конкретной
  причиной.
- Evidence записан в `feature_list.json` или `progress.md`.
- Репозиторий оставлен в состоянии, из которого следующая Codex-сессия может
  стартовать без восстановления контекста из чата.

## Завершение сессии (End of Session)

1. Запиши измененные файлы и verification evidence в `progress.md`.
2. Обнови статус активной фичи в `feature_list.json`.
3. Обнови `session-handoff.md`, если работа занимает несколько сессий.
4. Оставь конкретный следующий шаг, а не размытое напоминание.
