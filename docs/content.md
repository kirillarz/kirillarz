## 1. Правила использования этого файла

Этот файл — основной источник фактов для публичного сайта Кирилла Арзамасцева.
Визуальная концепция, структура блоков и требования к Lego-style интерфейсу
зафиксированы в `docs/site-brief.md`.

При работе с контентом необходимо:

* использовать только подтверждённые факты из этого файла;
* не придумывать опыт, технологии, количественные результаты, ссылки или достижения;
* не называть Кирилла экспертом, senior-специалистом или опытным product manager;
* не менять смысл утверждённых формулировок без необходимости;
* помечать отсутствующие ссылки и материалы как `TODO`, а не создавать фиктивные URL;
* не публиковать данные из разделов «Не выводить публично»;
* не выводить все технологии подряд: для карточек проектов использовать только наиболее важные.

# Проект 3. PM Simulator

### Статус

MVP готов.

### Период

2026 год.

### Формат

Индивидуальный проект для экзамена по программированию.

### Роль Кирилла

Автор идеи, геймдизайнер и разработчик.

### Краткое описание для карточки

> PM Simulator — 2D top-down игра на pygame, в которой игрок управляет IT-стартапом перед релизом: назначает задачи, помогает сотрудникам, решает кризисы и балансирует между скоростью, качеством, бюджетом и выгоранием команды.

### Цель игры

Показать в игровой форме типичный хаос IT-команды перед релизом и роль project manager, который не делает всю работу сам, а помогает команде двигаться к результату.

### Игровой процесс

Игрок:

* запускает игровую сцену из главного меню;
* перемещается по офису с помощью `WASD` или стрелок;
* ставит игру на паузу клавишей `Esc`;
* может продолжить игру или выйти в главное меню;
* видит офисную карту, рабочую зону, кухню, переговорку и канбан-зону;
* открывает канбан-доску;
* назначает задачи сотрудникам;
* подходит к сотрудникам и решает их проблемы;
* выбирает реакцию на кризисные ситуации;
* балансирует между скоростью разработки, качеством, бюджетом и выгоранием;
* может досрочно выпустить релиз после выполнения пула задач;
* завершает сессию победой или поражением;
* видит экран результата, счёт и рекорд.

### Система сотрудников

Сотрудники:

* имеют специализацию: backend, frontend, QA, DevOps и AI;
* автономно перемещаются по офису;
* выполняют задачи с разной скоростью;
* имеют риск ошибок;
* устают, отдыхают, испытывают стресс и могут выгореть;
* могут просить помощи при сложной задаче или приближающемся дедлайне.

### Технологии

* Python
* pygame
* объектно-ориентированное программирование
* игровая логика
* система состояний
* работа с событиями и UI
