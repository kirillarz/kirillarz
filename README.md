## О проекте

Это мой личный сайт-визитка и портфолио. Он создан, чтобы в одном месте рассказать обо мне, показать проекты, опыт работы в команде и интерес к разработке, продуктам и управлению.

Сайт задуман как не просто стандартное портфолио с карточками технологий, а как личное цифровое пространство с визуальной атмосферой небольшого IT-стартапа перед релизом.

Основная визуальная идея — **пиксельный офис в духе PM Simulator**: немного самоиронии про дедлайны, баги, канбан-доски и попытки привести рабочий хаос к понятному результату.

Проект также нужен мне как практический кейс: от проектирования интерфейса и работы с анимациями до подготовки качественного статического frontend-приложения, которое можно вручную опубликовать на выбранной платформе.

## Возможности MVP

* адаптивная главная страница с приоритетом на мобильные устройства;

* hero-блок с пиксельной офисной сценой;
* плавные анимации при скролле и взаимодействии с элементами;
* раздел с избранными проектами;

* возможность скачать резюме в PDF.

## Технологии

### Основной стек

* **Vite** — сборка, dev-сервер и базовая архитектура фронтенд-приложения;
* **React** — компонентный подход к разработке интерфейса;
* **TypeScript** — типизация и надёжность кода;
* **React Router** — маршруты главной страницы и страницы `/employer`;
* **CSS Modules** — компонентные стили;
* **глобальные CSS-файлы** — дизайн-токены, темы, базовая типографика и общие правила.

### Анимации и визуальная часть

* CSS-переходы и keyframes — простые hover-состояния, переключение темы и микро-анимации;
* SVG и CSS-композиции — временные тематические обложки и декоративные элементы;
* финальные изображения и сложные визуальные материалы добавляются только после отдельного согласования.

### Качество кода

* **TypeScript** — проверка типов после настройки проекта;
* **ESLint** — планируемая проверка качества кода;
* **Vitest** — планируемое тестирование полезной интерактивной логики после появления такой логики.

### Развёртывание

* **GitHub Actions** — автоматические проверки качества без деплоя;
* **ручная публикация** — деплой выполняется владельцем проекта после выбора платформы хостинга.

## Локальный запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/<your-username>/<repository-name>.git
cd <repository-name>
```

### 2. Установите зависимости

```bash
npm install
```

### 3. Запустите проект в режиме разработки

```bash
npm run dev
```

После запуска сайт будет доступен по адресу:

```text
http://localhost:5173
```

### 4. Соберите production-версию

```bash
npm run build
npm run preview
```

## Принципы проекта

* **Сначала понятность, потом эффектность.** Анимации и пиксельный стиль не должны мешать посетителю быстро понять, кто я и что умею.
* **Мобильная версия — не второстепенная.** Сайт проектируется mobile-first.
* **Без технологий ради технологий.** Сложные инструменты добавляются только когда они реально улучшают продукт.
* **Один сайт — разные сценарии.** Друзья могут посмотреть атмосферную главную, а работодатель — быстро найти проекты, навыки и резюме.
* **Контент важнее количества.** Лучше несколько сильных проектов с понятным описанием, чем десятки пустых карточек.
